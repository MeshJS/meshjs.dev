## Why Is Cardano Wallet Integration Complex?

Cardano wallet integration is complex because multiple providers (Eternl, Lace, Yoroi) implement CIP-30 differently. Unlike Ethereum's standardized `window.ethereum`, each Cardano wallet registers separately and varies in return formats, error handling, and connection persistence. The Mesh Cardano SDK normalizes these differences for consistent JavaScript Web3 development.

### What Is the CIP-30 Standard?

CIP-30 defines how Cardano dApps communicate with wallets through browser APIs:

- Register at `window.cardano.<walletName>`
- Provide `enable()` for connection
- Expose `getUtxos()`, `signTx()`, `submitTx()`
- Return consistent data structures

In theory, code for one wallet works with all. In practice, differences emerge.

### What Variations Exist Between Wallets?

| Issue | Description |
|-------|-------------|
| Method availability | Some wallets skip optional CIP-30 methods |
| Return formats | Hex strings vs byte arrays; address format varies |
| Error handling | "User declined" vs error code 2; inconsistent messages |
| Connection state | Some persist across reloads, others don't |
| Network detection | Getting mainnet/testnet works differently |

### How Do Cardano Wallets Differ from Ethereum?

```javascript
// Cardano: each wallet registers separately
window.cardano.eternl    // Eternl
window.cardano.lace      // Lace
window.cardano.yoroi     // Yoroi
window.cardano.typhon    // Typhon

// Ethereum: mostly standardized
window.ethereum          // Most wallets
```

New wallets emerge regularly, requiring code updates without a unified SDK.

### What User Experience Issues Occur?

- Users don't know which wallets are supported
- Cryptic connection failure errors
- Browser popup blocking
- Reconnection required after page refresh
- Multi-wallet users select wrong wallet

### How Does BrowserWallet Work?

```typescript
import { BrowserWallet } from "@meshsdk/core";

// Detect all installed wallets
const installedWallets = BrowserWallet.getInstalledWallets();
// [{ id: 'eternl', name: 'Eternl', ... }, ...]

// Connect to a specific wallet
const wallet = await BrowserWallet.enable("eternl");

// Unified API regardless of wallet

const address = await wallet.getChangeAddress();
const balance = await wallet.getBalance();
const network = await wallet.getNetworkId();
```

The Cardano SDK normalizes return values, handles missing methods gracefully, and provides consistent error types.

### How Does the useWallet Hook Work?

```tsx
import { useWallet } from "@meshsdk/react";

function TransactionButton() {
  const { connected, wallet, connect, disconnect } = useWallet();

  if (!connected) {
    return <button onClick={() => connect("eternl")}>Connect</button>;
  }

  return (
    <div>
      <button onClick={handleTransaction}>Send</button>
      <button onClick={disconnect}>Disconnect</button>
    </div>
  );

  async function handleTransaction() {
    const utxos = await wallet.getUtxos();
    // Build and submit transaction
  }
}
```

### How Do I Build Custom Wallet UI?

```tsx
import { BrowserWallet } from "@meshsdk/core";
import { useWallet } from "@meshsdk/react";

function CustomWalletSelector() {
  const { connect, connected, wallet } = useWallet();
  const installedWallets = BrowserWallet.getInstalledWallets();

  if (connected) return <ConnectedWalletDisplay wallet={wallet} />;

  return (
    <div className="wallet-grid">
      {installedWallets.map((w) => (
        <button key={w.id} onClick={() => connect(w.id)}>
          <img src={w.icon} alt={w.name} />
          <span>{w.name}</span>
        </button>
      ))}
    </div>
  );
}
```

## How Do I Handle Common Issues?

Common Cardano dApp development issues include connection persistence, network validation, popup blocking, and transaction errors. The Mesh TypeScript blockchain SDK provides patterns for handling each.

### How Do I Persist Wallet Connections?

```tsx
import { useEffect } from "react";
import { useWallet } from "@meshsdk/react";

function WalletManager() {
  const { connected, connect, wallet } = useWallet();

  useEffect(() => {
    const saved = localStorage.getItem("connectedWallet");
    if (saved && !connected) connect(saved);
  }, []);

  useEffect(() => {
    if (connected && wallet) {
      localStorage.setItem("connectedWallet", wallet.walletId);
    }
  }, [connected, wallet]);

  return <CardanoWallet />;
}
```

### How Do I Validate Network?

```tsx
import { useWallet } from "@meshsdk/react";

function NetworkGuard({ children, requiredNetwork }) {
  const { wallet, connected } = useWallet();
  const [networkId, setNetworkId] = useState(null);

  useEffect(() => {
    if (connected) wallet.getNetworkId().then(setNetworkId);
  }, [connected, wallet]);

  if (connected && networkId !== requiredNetwork) {
    return <div>Please switch to {requiredNetwork === 1 ? "Mainnet" : "Testnet"}</div>;
  }
  return children;
}
```

### How Do I Handle Popup Blocking?

```tsx
async function connectWallet(walletId) {
  try {
    await connect(walletId);
  } catch (error) {
    if (error.message.includes("popup")) {
      showNotification("Please allow popups to connect your wallet");
    } else {
      showNotification("Connection failed. Please try again.");
    }
  }
}
```

### How Do I Handle Transaction Errors?

```tsx
async function signAndSubmit(txBuilder) {
  try {
    const unsignedTx = await txBuilder.complete();
    const signedTx = await wallet.signTx(unsignedTx);
    const txHash = await wallet.submitTx(signedTx);
    return { success: true, txHash };
  } catch (error) {
    if (error.code === 2 || error.message.includes("User")) {
      return { success: false, error: "Cancelled by user" };
    }
    if (error.message.includes("insufficient")) {
      return { success: false, error: "Insufficient funds" };
    }
    return { success: false, error: "Transaction failed" };
  }
}
```

## How Do I Handle Advanced Scenarios?

Advanced Cardano dApp development scenarios include multi-wallet support, hardware wallet integration, and server-side wallets for backend operations. The Mesh Cardano SDK handles all these patterns.

### How Do I Support Multiple Wallets?

```tsx
const [wallets, setWallets] = useState([]);

async function addWallet(walletId) {
  const newWallet = await BrowserWallet.enable(walletId);
  setWallets(prev => [...prev, { id: walletId, instance: newWallet }]);
}
```

### How Do Hardware Wallets Work?

```tsx
// Hardware wallets work through browser extensions
// No special code needed - they appear in getInstalledWallets()
const wallets = BrowserWallet.getInstalledWallets();
// Includes Ledger, Trezor extensions when available
```

### How Do I Use Server-Side Wallets?

```typescript
import { MeshWallet, BlockfrostProvider } from "@meshsdk/core";

const provider = new BlockfrostProvider("<api-key>");

const wallet = new MeshWallet({
  networkId: 0,
  fetcher: provider,
  submitter: provider,
  key: {
    type: "mnemonic",
    words: ["your", "mnemonic", "words", "..."],
  },
});

// Same API as BrowserWallet
const address = wallet.getChangeAddress();
const utxos = await wallet.getUtxos();
```
